	repository. This seems to make more sense, the more so since this is also how
	upstream Slackware works. More generally, the extras/ repository is supposed
	to provide all those packages that are not part of a "classic"
	one-app-per-task MLED desktop, e. g. alternative or less used applications or
	simply specific packages requested by Microlinux clients. Note that if a
	package requires one or more dependencies, these will all be added to the
	main repository to spare you the hassle of hunting them down one by one.
